# FIEND.md

## The Fiend Studios Manifesto

> **What's the One Thing?**

Everything begins with that question.

---

# Purpose

Fiend Studios exists to turn meaningful ideas into finished products.

We do not collect projects.

We finish them.

Every story, application, brand, comic, AI workflow, automation, video, website, product, or business exists to strengthen Fiend Studios as a whole.

The studio is the product.

Everything else is proof that the studio can create.

---

# Mission

Build exceptional things.

Finish exceptional things.

Teach others through exceptional things.

---

# Philosophy

Ideas are infinite.

Time is not.

Execution is our competitive advantage.

Our value is not measured by how many ideas we have.

It is measured by how many ideas reach the finish line.

---

# The One Thing

Before beginning any work, ask:

> **What is the One Thing that most improves Fiend Studios today?**

If the answer is unclear, stop.

Find the answer before writing code, creating art, designing systems, or starting another project.

If multiple answers exist, choose the one with the greatest long-term leverage.

---

# Core Principles

## Finish before expanding.

A finished product creates more value than ten unfinished masterpieces.

---

## Build assets, not distractions.

Everything created should become one of the following:

* reusable knowledge
* reusable code
* reusable art
* reusable systems
* reusable reputation
* recurring revenue

If it becomes none of these, question whether it should exist.

---

## Simplicity wins.

Never add complexity simply because it is interesting.

Complexity must earn its place.

---

## Quality compounds.

Do work that people remember.

A single remarkable project creates more opportunity than dozens of average ones.

---

## Systems should disappear.

The best workflow is one that becomes invisible.

Never create software simply to manage software.

Never automate work that should not exist.

Never build a system that creates another system to maintain.

---

# Active Project Rule

There are only three categories.

## NOW

The current flagship objective.

Everything else supports this.

---

## NEXT

Projects approved for the following cycle.

Maximum of three.

---

## VAULT

Every other idea.

Ideas are never deleted.

Ideas are not obligations.

Capturing an idea does not make it active.

---

# Decision Filter

Every significant decision should answer these questions.

Does this make Fiend Studios stronger?

Does this create long-term value?

Can this realistically be finished?

Will someone care that this exists?

Does this improve our reputation?

If this means saying yes...

What must we intentionally say no to?

---

# AI Collaboration

Every AI working inside Fiend Studios should follow these principles.

Do not optimize for novelty.

Optimize for usefulness.

Do not impress with complexity.

Impress with clarity.

Challenge assumptions respectfully.

Identify risks honestly.

Protect existing work.

Prefer evidence over confidence.

When uncertain, say so.

When multiple good options exist, explain the tradeoffs.

When work is complete, stop.

---

# Brand Standards

Everything published should make people think:

"They really care about quality."

Every release represents the studio.

Never ship work that weakens the brand.

---

# Definition of Finished

Finished means:

It works.

It is tested.

It is documented.

It is maintainable.

It is something we are proud to attach the Fiend Studios name to.

Finished does not mean perfect.

Finished means valuable.

---

# Weekly Review

Once each week ask:

What did we finish?

What created the most value?

What should move into the Vault?

What deserves to become the One Thing next week?

---

# Final Principle

Fiend Studios is not a collection of projects.

It is a promise.

A promise that whatever carries its name was built with intention, finished with discipline, and exists because it mattered.

Whenever uncertainty appears, return to one question.

> **What's the One Thing?**
